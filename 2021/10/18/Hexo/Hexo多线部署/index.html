Hexo多线部署

对比五种静态网页托管服务

网页托管服务很多，如 Github Pages、Gitee Pages、Coding 持续集成、Cloudflare Pages 、Vercel等……

Coding、Gitee 为国内服务提供商，需要实名认证，其中 Coding 和鹅云联系密切，需要 超大量 的认证工作，没个三五天搞不完。Cloudflare Pages、Github Pages、Vercel 为国外服务提供商，相对而言访问速度较慢。

壹丨网页托管服务部署对比

1. Coding

部署过程繁琐，需要配置鹅云权限、各种实名认证

优点缺点
速度快
可被百度爬取		需要认证的太多
构建过程漫长

Github Pages

优点缺点
部署简单
构建速度快		访问速度慢
不能被百度收录

Cloudflare Pages

优点缺点
部署简单
部署&自定义域名一条龙
速度还行		不能百度收录

Gitee Pages

优点缺点
部署简单上传后需要手动部署
不能避免审查

Vercel

优点缺点
部署简单
自带域名解析		不能百度收录

贰丨双线部署方法（已放弃）

双线部署，国内IP访问国内源站，国外IP访问国外源站。

以Coding和Cloudflare Pages为例：

第一步，各自部署并生成各自的网页链接；

第二步，在阿里云解析中设置CNAME记录，为Coding设置智能线路，为Cloudflare Pages设置为国外IP线路。
